# SaaS UI Overhaul — 设计文档

**日期**: 2026-05-29
**状态**: 待实现
**分支**: `saas-ui-overhaul`
**设计风**: 靛蓝紫玻璃 Indigo Glass — 双模式

## 目标

全系统 UI 从移动端延伸风格改造为专业 SaaS 企业后台风格。参考 Linear / Vercel / Stripe 布局模式 + Apple 玻璃质感。

## 配色方案

## Dashboard 设计 — 科技感数据概览

### 设计理念

不以彩色大卡片为主导。SaaS dashboard = 数据优先+克制配色+空间节奏。参考 Stripe / Vercel Analytics / Linear 的 dashboard 语言。

### 布局

```
┌─────────────────────────────────────────────┐
│ 首页 › 财务                                  │
│                                             │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐     │
│ │ 营业收入  │ │ 应收账款  │ │ 回款率    │     │
│ │ ¥12.8M  │ │ ¥4.2M   │ │ 67.3%    │     │
│ │ ↑8.2%   │ │          │ │          │     │
│ └──────────┘ └──────────┘ └──────────┘     │
│                                             │
│ ┌─────────────────────┐ ┌────────────────┐  │
│ │ 12月收入趋势 (图表)   │ │ 预算执行率 (进度) │  │
│ └─────────────────────┘ └────────────────┘  │
│                                             │
│ ┌──────────────────────────────────────┐    │
│ │ 待处理事项: 3 票据 / 2 待收款 / 1 审批  │    │
│ └──────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```

### KPI 指标卡

- **Light**: 玻璃半透明卡片 + 微边框 + 数字大字号(28px) + 趋势箭头
- **Dark**: 纯深炭卡片 + 细边框 + 数字大字号
- 每个 KPI 下方可选：趋势箭头(↑↓) + 百分比变化
- 最大 3-4 个 KPI，不堆砌

### 图表

- fl_chart LineChart：靛蓝紫渐变线 + 浅色填充
- 去掉图表容器的边框/背景，融入页面
- X/Y 轴标签低调灰色

### 预算进度

- 分类色横条（保留现有功能）
- 去掉多余容器，直接展示

### 通用 Dashboard 模式

所有模块 Dashboard（首页/HR/市场/招投标/PM）都遵循同一套视觉语言：
- 指标卡 + 图表 + 快捷入口
- 不用彩色大渐变卡片
- 玻璃质感(Light) / 纯色细边框(Dark)
- 字体保持 Microsoft YaHei

### 3. 页面内 Tab 替代全屏切换

所有模块 Dashboard（财务/HR/市场/招投标/项目管理）:
- 面包屑导航：`首页 › 模块名`
- TabBar 切换子页面（Dashboard / 票据 / 预算 / 支出 / 凭证）
- 去掉 Navigator.push 全屏切换
- 去掉子页面的 AppBar 返回按钮

### 4. 数据表格化

以下列表从卡片堆叠改为紧凑 DataTable：
- 票据列表
- 支出列表
- 凭证列表
- 客户列表
- 项目列表

### 5. Dashboard 适配

财务 Dashboard 适配新配色：
- KPI 卡片：玻璃质感（Light）/ 纯色细边框（Dark）
- 收入折线图：靛蓝紫线
- 预算进度条：分类色

## 实施优先级

1. 主题配色（AppTheme 重写）— 最先，所有后续依赖
2. 侧边栏重构（ResponsiveScaffold）— 全局布局基础
3. Dashboard 适配（财务）— KPI卡片重做
4. 页面内 Tab（财务模块）— 去掉全屏切换
5. 数据表格化（票据页）— 卡片→DataTable
6. 扩展到其他模块 — 市场/招投标/PM/HR

## 不改变

- 后端 API — 零改动
- 业务逻辑 — 零改动  
- 数据模型 — 零改动
- 文件预览系统 — 不变
