Step 4: 后端增强 + 总验证

KnowledgeQARequest加 history: list[dict] = [] 字段
/knowledge/qa 端点history 拼入 system prompt，增强搜索词，返回 sources 加 content_preview
全模块联调手动验证核心链路 + E2E 回归

为什么这个顺序

  • 3.1→3.2 的页面简单（表单+展示），可以快速产出建立节奏
  • 3.3 合同中心最复杂，放后面有足够的上下文和 pattern 参考
  • 后端增强放最后，避免改 API 影响前端联调

风险

  • ContractDetail 的版本历史 Tab 需要对现有 ContractTab 做少量改动（加跳转入口）
  • ContractDiff 的分屏 UI 在移动端需要降级为内联模式
